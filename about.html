✈️ About archive.aero

Nearly a century of U.S. aeronautical charts, scrubbable like a time machine.

What is this?

archive.aero is a free viewer for historical FAA VFR Sectional Aeronautical Charts. Drag the timeline and watch airspace, airports, navaids, and cartography evolve — from the earliest airway strip charts of the 1930s to the edition in effect today. Individual chart sheets are stitched into a single seamless mosaic for each edition, so you can pan the whole country at any point in time.

⚠️ Historical use only

Every chart on this site is a historical scan and is not current. Do not use these charts for navigation. Always consult official, in-date FAA publications for flight planning and operations.

How it works

There is no tile server and no application backend. A data pipeline turns archival scans into immutable single-file tile archives, and the site streams tiles straight out of edge storage.

NARA / FAA 1. Source

Historical charts are digitized scans held by the U.S. National Archives (Record Group 237 — Records of the FAA). Current editions come from FAA digital chart products.

↓
Python + GDAL 2. Georeference & mosaic

Each scan is georeferenced, its paper collar cropped, warped to Web Mercator, and mosaicked with its neighbors into one nationwide GeoTIFF per edition date — across decades of changing projections, scales, and cartographic conventions.

↓
Go 3. Tile

A custom converter slices each mosaic into WebP map tiles and packs them into a single PMTiles archive — one immutable file per edition, ordered for efficient range reads.

↓
Cloudflare R2 + Workers 4. Serve

Archives live in object storage at the edge. A small Worker proxies HTTP range requests, adds caching and CORS, and logs anonymous usage — no database, no server to maintain.

↓
Leaflet + vanilla JS 5. View

The site itself is a static page. The browser reads tiles directly from the PMTiles archives using range requests, double-buffers chart layers so scrubbing never flickers, and manages tile memory carefully enough to run smoothly on an iPad.

Engineering notes

  • Flicker-free scrubbing — when you change dates, the outgoing edition stays on screen until the incoming one has rendered, so the basemap never flashes through.
  • Memory-aware caching — decoded tiles are held in LRU caches with backpressure, with tighter limits and decoder fallbacks for iOS Safari.
  • Prefetching — neighboring editions load in the background while you scrub, so stepping through time feels instant.
  • Keyboard friendly — ← → to step editions, Space to play, ? for all shortcuts.

The whole project — frontend, Worker, and the chart-processing pipeline — is open source on GitHub.

Who built this?
